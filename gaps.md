# 当前缺失的技能和应对方案

## 缺失总览

| 缺失 | 严重程度 | 应对方案 |
|------|---------|---------|
| 微信 H5 JS-SDK | 🟡 中 | 需自建，见 `skills/08-wechat-h5.md`，降级方案 Web Share API |
| 前端埋点自动化 | 🟡 中 | 手动集成 PostHog/GA4 JS SDK |
| 微信公众号权限 | 🟡 中 | 降级为 Web Share API + 复制链接 |

## 详细说明

### 微信 H5 JS-SDK（裂变功能）

**为什么重要：** 微信内分享需要 JS-SDK 才能自定义分享卡片（标题+描述+图片），否则只能复制链接。

**应对方案：**
1. `skills/08-wechat-h5.md` 已提供完整的实现模板（前端+后端）
2. Agent 可以直接按模板编码
3. 但必须等人类提供微信公众号 AppID/AppSecret 和配置 JS 安全域名

**降级方案：** 先用浏览器原生 Web Share API 兜底，微信内用户可复制链接分享。

### 前端埋点自动化

**为什么不算严重：** 埋点本身就是几行 JS 代码，不需要 skill 自动化。关键是知道**埋什么**，而不是**怎么埋**。

**应对方案：**
1. 参考 `skills/07-analytics.md` 中的必埋点列表
2. 用 PostHog JS SDK 手动集成（代码量极少）
3. 上线后用 GA4 MCP 分析数据

## 需要人类提供的资源清单

以下是 Agent 无法自行获取的资源。如果缺失，Agent 应写入 `BLOCKED.md` 并继续做其他不依赖这些资源的工作。

| 资源 | 用于 | 状态 |
|------|------|------|
| 服务器信息 | 部署 | 在 `.env.dev` 中配置 |
| Stripe API Key (test) | 支付测试 | 在 `.env.dev` 中配置 |
| Stripe Webhook Secret | Webhook 验签 | 在 `.env.dev` 中配置 |
| 微信公众号 AppID | 微信分享 | ⬜ 未获取（降级方案可用） |
| 微信公众号 AppSecret | 微信分享 | ⬜ 未获取（降级方案可用） |
| GA4 / PostHog | 数据分析 | Agent 自行注册 |
