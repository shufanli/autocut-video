# PRD: AI 口播视频自动剪辑 — MVP v1.0

## 1. 产品概述

**产品定位：** 口播视频的 AI 剪辑师——用户上传原始素材，多 Agent 协作完成专业级剪辑，输出可直接发布的成品视频。

**一句话描述：** 上传口播素材 → AI 自动去口误、加字幕 → 输出可直接发布的成品视频。（v1.1 追加：内容理解、智能配乐、AI 特效）

**与竞品的本质区别：**

| | 剪映 | Descript | 我们 |
|--|------|---------|------|
| 核心体验 | 用户手动操作，AI 辅助 | 编辑文字稿 = 编辑视频 | 用户上传素材，AI 端到端出片 |
| AI 角色 | 工具（字幕、模板） | 工具（转录、填充词删除） | **剪辑师**（理解内容，做创意决策） |
| 用户需要学什么 | 时间线、关键帧、转场… | 文本编辑 | 几乎不需要学 |

**核心设计哲学：**
1. **素材进，成品出（Zero-Edit Promise）**：用户唯一的操作是上传素材和选择偏好，不需要理解时间线、关键帧、转场等任何剪辑概念。AI 做所有决策，用户只做"确认/微调"。
2. **语义驱动，非模板驱动**：特效、配乐、节奏不是套模板，而是 LLM 理解内容语义后做出的创意决策。同一句"很多人都搞错了"在知识科普和情绪吐槽中会触发不同的特效。
3. **机械层先行，创意层渐进**：MVP 优先解决确定性高的机械工作（去口误、字幕、合并），创意层（配乐、特效）作为增值体验，但不阻塞核心价值交付。
4. **不确定即可配置**：所有涉及审美判断的参数（特效密度、停顿阈值、填充词列表、定价）均设为可配置项，通过用户数据迭代收敛。
5. **优雅降级**：LLM 不可用时系统仍能输出基础版成品，核心价值不依赖单一外部服务。

**目标用户：**
- 口播日更/周更的内容创作者（知识博主、个人 IP、微商）
- 有大量分段素材需要拼接
- 技术水平偏小白，不想学复杂工具
- 要求"量大够用"而非"极致精美"

## 2. 用户痛点

| 痛点 | 严重程度 | 现有解决方案 | 成本 |
|------|---------|-------------|------|
| 口误/停顿需手动逐个删除 | 高 | 剪映手动裁剪 | 30-60 分钟/条 |
| 不知道怎么分段、加节奏 | **极高** | 靠经验或招剪辑师 | 必须招人 |
| 字幕需要校对和调样式 | 中 | 剪映 AI 字幕 | 20-30 分钟/条 |
| 不知道配什么背景音乐 | 高 | 手动从音乐库挑选 | 15-30 分钟/条 |
| 视频平淡没吸引力 | **极高** | 学特效或招人 | 必须招人 |
| 多段素材拼接排序 | 中 | 手动导入排列 | 20-40 分钟/条 |
| 综合：一条视频从录完到发出 | — | 以上全部 | **1-3 小时/条 或 200-500 元外包** |

**核心洞见：** 前三个痛点是"体力活"，AI 规则引擎就能解决；后三个痛点是"脑力活"——需要审美判断，这是大模型的真正价值所在，也是和剪映 AI 的本质差异。

## 3. 系统架构：多 Agent 协作

### 3.1 架构概览

系统采用多 Agent 协作模式，每个 Agent 负责剪辑流程中的一个专业环节。Agent 之间通过结构化 JSON 传递数据，由编排层（Orchestrator）协调执行顺序。

```
┌──────────────────────────────────────────────────────────────┐
│                    Orchestrator（编排层）                       │
│         接收用户素材和偏好，调度各 Agent，汇总输出                │
└──────┬──────┬──────────┬──────────┬──────────────────────────┘
       │      │          │          │
       ▼      ▼          ▼          ▼
   素材合并  语音识别   口误清理     字幕          ┌─ v1.1 ──────────────┐
   Agent    Agent     Agent       Agent         │  内容理解 Agent (LLM) │
   (规则)   (Whisper)  (规则层)    (规则)          │  配乐 Agent (LLM)     │
                                                │  特效 Agent (LLM+规则) │
                                                │  口误清理 LLM 层       │
                                                └────────────────────────┘
```

### 3.2 Agent 职责定义

#### Agent 1: 素材合并 Agent（规则驱动）
- **输入：** 用户上传的 1~N 个视频文件 + 排序
- **输出：** 合并后的单个视频文件
- **实现：** FFmpeg concat demuxer，纯规则，不调用 LLM

#### Agent 2: 语音识别 Agent（模型驱动）
- **输入：** 视频文件
- **输出：** 带 word-level 时间戳的文字稿 JSON
  ```json
  [{"text": "大家好", "start_ms": 0, "end_ms": 800, "confidence": 0.95}, ...]
  ```
- **实现：** Whisper large-v3 API
- **语言：** MVP 仅支持普通话

#### Agent 3: 内容理解 Agent（LLM 驱动）⭐ 核心差异化 —— ⚠️ v1.1，不在 MVP 范围
- **输入：** 完整文字稿
- **输出：** 内容分析 JSON
  ```json
  {
    "summary": "这是一条关于时间管理的知识分享视频",
    "segments": [
      {"title": "开场引入", "start_ms": 0, "end_ms": 15000, "type": "hook"},
      {"title": "痛点描述", "start_ms": 15000, "end_ms": 45000, "type": "problem"},
      {"title": "方法一：番茄钟", "start_ms": 45000, "end_ms": 120000, "type": "solution"},
      ...
    ],
    "key_moments": [
      {"text": "这是最关键的一点", "ms": 78000, "type": "emphasis"},
      {"text": "很多人都搞错了", "ms": 135000, "type": "surprise"}
    ],
    "mood": "informative_casual",
    "suggested_pace": "medium"
  }
  ```
- **LLM 职责：**
  - 理解视频的内容结构（开场→论点→论据→总结）
  - 识别关键时刻（金句、转折、高潮）
  - 判断整体情绪基调（轻松/严肃/激动/教学）
  - 建议整体节奏（快/中/慢）
- **实现：** 调用 LLM（Claude / GPT-4o），Prompt 包含文字稿全文 + 角色设定（"你是一个专业的短视频剪辑师"）

#### Agent 4: 口误清理 Agent（MVP 仅规则层；LLM 层 v1.1）
- **输入：** 文字稿 JSON（MVP）；+ 内容分析（v1.1）
- **输出：** 口误标记列表
  ```json
  [
    {"type": "filler", "start_ms": 3200, "end_ms": 3800, "action": "delete"},
    {"type": "repeat", "start_ms": 12000, "end_ms": 13500, "action": "keep_last"},
    {"type": "restart", "start_ms": 25000, "end_ms": 28000, "action": "delete", "reason": "说错后重新开始"},
    {"type": "long_pause", "start_ms": 45000, "end_ms": 47500, "action": "shorten_to_500ms"}
  ]
  ```
- **规则层（不调用 LLM）：**
  - 重复词/短语检测（连续相同 token）
  - 填充词检测（嗯、啊、那个、就是，列表可配置）
  - 长停顿检测（>1.5s，阈值可调）
- **LLM 层（调用 LLM）：**
  - 错误后重说检测（"把这个...不对，把那个"→ 删除"把这个...不对"）
  - 结合内容理解，判断某些停顿是"思考停顿"还是"无意义停顿"（如演讲中的戏剧性停顿应保留）

#### Agent 5: 配乐 Agent（LLM 驱动）⭐ 新增 —— ⚠️ v1.1，不在 MVP 范围
- **输入：** 内容分析 JSON（mood、segments、pace）
- **输出：** 配乐方案
  ```json
  {
    "bgm": {
      "track_id": "upbeat_knowledge_01",
      "volume": 0.15,
      "fade_in_ms": 2000,
      "fade_out_ms": 3000
    },
    "segment_music": [
      {"segment": "hook", "volume_boost": 0.05, "reason": "开场需要更有能量感"},
      {"segment": "solution", "volume_boost": -0.03, "reason": "讲解部分降低音乐避免干扰"}
    ]
  }
  ```
- **LLM 职责：**
  - 根据内容情绪（mood）从预置音乐库中选择匹配的 BGM
  - 根据内容段落调整音量（讲重点时降低、转场时提升）
  - 决定淡入淡出时机
- **音乐库：** MVP 内置 20-30 首免版权 BGM，按情绪分类（轻松/严肃/激动/温暖/科技感）
- **实现：** LLM 选曲 + FFmpeg 混音

#### Agent 6: 字幕 Agent（规则驱动 + LLM 辅助）
- **输入：** 文字稿 + 内容分析（key_moments）
- **输出：** 字幕渲染指令
  ```json
  [
    {"text": "大家好", "start_ms": 0, "end_ms": 800, "style": "normal"},
    {"text": "这是最关键的一点", "start_ms": 78000, "end_ms": 80000, "style": "emphasis", "highlight_words": ["最关键"]}
  ]
  ```
- **规则层：** 基于 Whisper 时间戳生成逐句字幕
- **LLM 辅助：** 利用内容理解 Agent 的 key_moments，自动对关键词/金句使用高亮样式
- **样式预设：**
  - 简洁白字（默认）
  - 黑底白字（适合画面复杂场景）
  - 彩色高亮（关键词自动变色加粗）
- **用户可调：** 字体大小（小/中/大）、位置（上/中/下）、是否显示

#### Agent 7: 特效 Agent（LLM + 规则混合）⭐ 新增 —— ⚠️ v1.1，不在 MVP 范围
- **输入：** 内容分析 JSON（segments、key_moments）
- **输出：** 特效指令列表
  ```json
  [
    {"type": "zoom_in", "target_ms": 78000, "duration_ms": 500, "scale": 1.15, "reason": "强调关键观点"},
    {"type": "shake", "target_ms": 135000, "duration_ms": 300, "intensity": "light", "reason": "配合'很多人搞错了'的惊讶感"},
    {"type": "transition", "at_ms": 45000, "style": "fade", "duration_ms": 500, "reason": "段落切换"},
    {"type": "lower_third", "start_ms": 45000, "end_ms": 120000, "text": "方法一：番茄钟", "reason": "段落标题"}
  ]
  ```
- **LLM 职责：**
  - 基于 key_moments 决定在哪里加特效（不是随机撒特效，是语义驱动）
  - 选择特效类型和强度（重点→放大，惊讶→抖动，转场→淡入淡出）
  - 在段落切换处自动加转场
  - 为每个内容段落生成 lower-third 标题条
- **MVP 特效库（基础款，FFmpeg 可实现）：**
  - `zoom_in` — 画面放大（强调）
  - `zoom_out` — 画面缩小（总结/远景）
  - `shake` — 轻微抖动（惊讶/强调）
  - `fade` — 淡入淡出（转场）
  - `lower_third` — 底部标题条（段落标记）
  - `flash` — 白闪（节奏点）
- **约束：** 特效密度由 LLM 控制，每分钟不超过 3 个特效（避免过度花哨）

### 3.3 编排层（Orchestrator）

**MVP 执行流程：**
```
用户上传素材
  │
  ▼
Agent 1: 素材合并（如有多段）
  │
  ▼
Agent 2: 语音识别（Whisper）
  │
  ├──────────────────┐
  ▼                  ▼
Agent 4: 口误清理   Agent 6: 字幕生成
（规则层）           （规则层）
  │                  │
  └────────┬─────────┘
           ▼
     用户预览与确认
           │
           ▼
     FFmpeg 渲染合成（剪切 + 字幕拼接）
           │
           ▼
         视频导出
```

**v1.1 扩展流程（在 Agent 2 之后增加 LLM 创意层）：**
```
Agent 2 → Agent 3 内容理解（LLM） → [并行] Agent 4 LLM 层 / Agent 5 配乐 / Agent 7 特效
```

**MVP 并行优化：**
- Agent 4（口误清理规则层）和 Agent 6（字幕）可并行执行
- MVP 零 LLM 调用，纯规则驱动，延迟可控

**用户可配置的偏好（上传时选择）：**

MVP 默认模式为"AI 自动决定"——用户无需做任何选择，系统使用默认参数处理。点击"高级设置"可展开以下选项：

| 偏好项 | 选项 | 默认值 | MVP / v1.1 |
|--------|------|--------|------------|
| 字幕样式 | 简洁白字 / 黑底白字 / 彩色高亮 | 简洁白字 | MVP |
| 字幕位置 | 上 / 中 / 下 | 下 | MVP |
| 视频风格 | 知识干货 / 情绪共鸣 / 轻松日常 / 带货种草 | 知识干货 | v1.1 |
| 节奏感 | 快节奏 / 中等 / 慢节奏 | 中等 | v1.1 |
| 特效强度 | 无特效 / 轻度 / 适中 | 轻度 | v1.1 |
| 是否配乐 | 是 / 否 | 是 | v1.1 |

MVP 偏好仅影响字幕 Agent；v1.1 偏好会注入到 Agent 3/5/7 的 LLM Prompt 中。

## 4. 功能需求

### 4.1 核心流程

```
MVP：上传素材 → 素材合并 → 语音识别 → [并行] 口误清理(规则) + 字幕生成(规则)
     → 用户预览确认（逐条审核口误） → FFmpeg 渲染（剪切+字幕） → 导出

v1.1：…→ 语音识别 → 内容理解(LLM) → [并行] 口误LLM层 + 配乐 + 字幕 + 特效 → …
```

### 4.2 功能清单

#### P0 — MVP 必须有（机械层，纯规则驱动，4-6 周交付）

**F1: 素材上传**
- 支持上传 1~N 个视频文件，拖拽排序
- 默认"AI 自动决定"模式，高级设置折叠（MVP 仅字幕样式/位置可调）
- 输入限制：MP4/MOV/WebM，≤1080p，单文件 ≤500MB，总时长 ≤30 分钟

**F2: 去口误（规则层）**
- 重复词、填充词（可配置列表）、长停顿自动检测与删除
- 用户可在预览页逐条审核每处口误的删除/保留

**F3: 自动字幕**
- 基于 Whisper 时间戳生成逐句字幕
- 3 套预设样式（简洁白字 / 黑底白字 / 彩色高亮）
- 用户可调字体大小、位置

**F4: 用户预览与确认**
- 文字稿预览：口误标红，每处可点击切换删除/保留
- 字幕样式预览
- 底部统计："共检测到 N 处口误，预计缩短 M 秒"
- 确认后进入渲染

**F5: 视频渲染与导出**
- FFmpeg 合成：剪切 + 字幕 → 单次渲染输出
- 输出格式：MP4（H.264 + AAC）
- 保持原始分辨率和比例

**F6: 用户认证与支付**
- 手机号 + 短信验证码登录（无需密码）
- 支付宝/微信支付接入

**F7: 基础内容审核**
- 上传阶段接入内容安全 API（如阿里云内容安全），对视频帧抽样审核
- 敏感词过滤（文字稿阶段）
- 人工抽检通道（后台标记可疑内容供运营审核）

**F8: 反作弊**
- 设备指纹（device fingerprint）识别，同一设备限制注册账号数量（默认 2 个）
- 同一 IP 每小时发送验证码上限（默认 10 次）
- 发送验证码前增加图形验证码/滑块验证码
- 接码平台常用号段识别与拦截

#### P1 — v1.1（LLM 创意层 + 增值功能）

- **内容理解 Agent（LLM）：** 分析内容结构、识别关键时刻、判断情绪基调
- **智能配乐 Agent（LLM）：** 根据内容情绪自动选曲、分段音量调节、内置 20-30 首免版权 BGM（来源：Pixabay / Free Music Archive CC0 音乐，保留授权证明）
- **特效 Agent（LLM）：** 语义驱动的 zoom_in / shake / fade / lower_third / flash
- **口误清理 LLM 层：** 错误后重说检测、戏剧性停顿保留
- **偏好系统完整版：** 视频风格、节奏感、特效强度、配乐开关
- **批量任务队列：** 支持一次上传多组素材（最多 5 组），系统依次处理，全部完成后一键打包下载（高频用户刚需）
- 多比例导出（9:16 + 16:9）
- OpenClaw Skill 封装
- 自定义字幕模板
- 用户上传自己的 BGM
- 更多特效（画中画、分屏、动态贴纸）

#### P2 — 远期

- 风格模板市场（社区共享/交易）
- 自然语言剪辑指令（"把节奏加快""这段要更有感染力"）
- 课程/教程视频支持
- 电商产品视频支持

### 4.3 异常场景

| 场景 | 处理方式 |
|------|---------|
| 上传格式/大小不支持 | 前端校验 + 后端二次校验，返回明确错误 |
| 语音识别质量差（>30% 低置信度）| 告警用户，建议重录或手动校对 |
| LLM 调用失败/超时（v1.1） | 降级到纯规则模式（跳过内容理解、配乐用随机推荐、不加特效）。MVP 不调用 LLM，无此风险 |
| FFmpeg 渲染失败 | 重试 1 次；仍失败则通知用户，保留中间产物 |
| 处理总时长超时（>15 分钟）| 终止任务，通知用户 |

**LLM 降级策略（v1.1 生效）：** 当 LLM 不可用时，系统仍能输出"基础版"成品（去口误 + 字幕 + 无配乐无特效），确保核心功能不依赖 LLM 可用性。MVP 本身即为纯规则模式，不依赖 LLM。

## 5. 用户旅程

```
首次使用（MVP）：
  打开网站 → 手机号+验证码登录
  → 上传视频（多文件拖拽排序），偏好默认"AI 自动决定"
  → 等待 AI 处理（进度条：合并素材 → 语音识别 → 检测口误&生成字幕）
  → 预览页面：
      - 左侧：文字稿（口误标红，可逐条审核删除/保留）
      - 右侧：字幕样式设置
  → 确认 → 渲染 → 下载成品

老用户快速模式：
  上传 → 使用上次偏好 → 跳过预览直接渲染 → 下载
```

## 6. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 10 分钟视频端到端处理 ≤8 分钟（含 LLM 调用、渲染） |
| **LLM 延迟** | 单次 LLM 调用 ≤30 秒 |
| **并发** | MVP 阶段支持 3-5 个同时处理任务（CPU 服务器 worker 上限），超出排队，页面显示排队位置 |
| **可用性** | 99% uptime |
| **降级能力** | LLM 不可用时自动降级到纯规则模式，核心功能不中断 |
| **数据安全** | 视频 24 小时内删除；上传页面明确告知数据流向 |
| **隐私** | 不存储用户视频用于训练；文字稿发送至 LLM 时脱敏处理 |

## 7. 输入输出规格

**输出：**
- MP4（H.264 + AAC），原始分辨率和比例
- MVP 包含：去口误 + 字幕
- v1.1 追加：BGM + 基础特效

## 14. 指标体系与埋点方案

### 14.1 北极星指标

**成功下载视频数 / 周**（Successfully Downloaded Videos per Week）

理由：这个指标同时衡量了用户获取（注册）、核心价值交付（处理完成）、用户满意（愿意下载使用）三个环节。仅"处理完成"不代表用户满意，仅"付费"不代表产品好用（可能是冲动消费）。

### 14.2 过程指标

| 层级 | 指标 | 计算方式 | 目标值（MVP，可配置） |
|------|------|---------|---------------------|
| 获客 | 周新增注册用户数 | COUNT(users WHERE created_at in week) | ≥50 |
| 获客 | 注册→首次上传转化率 | 首次上传用户数 / 新注册用户数 | ≥60% |
| 激活 | 上传→处理完成率 | 处理完成任务数 / 开始处理任务数 | ≥90% |
| 激活 | 处理完成→下载转化率 | 下载任务数 / 处理完成任务数 | ≥70% |
| 留存 | 周活跃用户数（WAU） | 当周有上传行为的用户数 | — |
| 留存 | 次周留存率 | 上周活跃用户中本周仍活跃的比例 | ≥30% |
| 变现 | 付费转化率 | 付费用户数 / 免费额度用尽用户数 | ≥15% |
| 变现 | ARPU（月） | 月总收入 / 月活跃付费用户数 | ≥¥50 |
| 质量 | 平均处理时长 | AVG(completed_at - created_at) | ≤8 分钟（10 分钟视频） |
| 质量 | 处理失败率 | 失败任务数 / 总任务数 | ≤5% |
| 质量 | LLM 降级比例 | 降级任务数 / 总任务数 | ≤10% |
| 质量 | 用户满意度（👍比例） | 👍数 / (👍+👎) | ≥80% |

## 16. 非目标（MVP 明确不做的事情）

以下功能和场景在 MVP 阶段明确排除，避免范围蔓延：

1. **不做视频时间线编辑器**：用户不需要理解时间线概念，所有编辑通过文字稿和列表完成
2. **不做方言支持**：MVP 仅支持普通话，方言留到用户数据验证需求后再做
3. **不做 4K 视频支持**：最大 1080p，4K 增加处理成本和时间
4. **不做多比例导出**：MVP 只输出原始比例，9:16 和 16:9 转换留到 v1.1
5. **不做实时预览**：用户在预览页看文字稿和配置列表，不提供实时视频预览播放（渲染后才能看成品视频）
6. **不做用户自定义 BGM 上传**：MVP 仅使用预置音乐库
7. **不做推送通知**：处理/渲染完成后不发短信/微信通知，用户需回到页面查看（MVP 通过页面轮询实现）
8. **不做团队/多账号功能**：MVP 只支持个人账号
9. **不做 API/SDK 开放平台**：不开放给第三方调用（留到 Approach C 阶段）
10. **不做 OpenClaw Skill**：v1.0 只做 Web App，OpenClaw 封装留到 v1.1
11. **不做风格模板市场**：社区共享/交易模板是远期功能
12. **不做自然语言剪辑指令**：如"把节奏加快"，需要更复杂的交互和 LLM 能力
13. **基础内容审核**：接入内容安全 API（如阿里云内容安全）对上传视频帧抽样审核，文字稿阶段敏感词过滤，后台保留人工抽检通道。不做深度内容理解级审核
14. **不做国际化**：MVP 界面和文案仅中文
