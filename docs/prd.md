# PRD: AI 口播视频自动剪辑 — MVP v1.0

## 1. 产品概述

**产品定位：** 口播视频的"自动驾驶"——用户上传原始素材，AI 全自动输出可发布的成品视频。

**一句话描述：** 上传口播素材 → 自动去口误、加字幕、合并分段 → 几分钟出成品，零剪辑基础即可使用。

**目标用户：**
- 口播日更/周更的内容创作者（知识博主、个人 IP、微商）
- 有大量分段素材需要拼接
- 技术水平偏小白，不想学复杂工具
- 要求"量大够用"而非"极致精美"

## 2. 用户痛点

| 痛点 | 现有解决方案 | 成本 |
|------|-------------|------|
| 口播录音磕磕绊绊，需手动去口误 | 剪映手动剪辑 | 1-3 小时/条 |
| 需要加字幕 | 剪映 AI 字幕（仍需手动校对调整）| 30-60 分钟/条 |
| 多段素材需要拼接 | 手动导入排列 | 20-40 分钟/条 |
| 不想自己剪 | 外包剪辑师 | 200-500 元/条 |

**核心洞见：** 口播视频的剪辑流程高度标准化（去口误→字幕→拼接→导出），完全可被自动化。现有工具（剪映）是"辅助驾驶"——仍需用户手动操作；我们做"自动驾驶"——素材进、成品出。

## 3. 功能需求

### 3.1 核心流程

```
上传素材 → [合并分段] → 语音识别 → 口误检测 → 用户预览确认 → 剪切拼接 → 加字幕 → 导出
```

### 3.2 功能清单

#### P0 — MVP 必须有

**F1: 素材上传与合并**
- 支持上传 1~N 个视频文件
- 支持拖拽排序，确定合并顺序
- 自动合并为单个视频后进入处理流程
- 输入限制：MP4/MOV/WebM，≤1080p，单文件 ≤500MB，总时长 ≤30 分钟

**F2: 自动去口误**
- 自动检测并标记以下 3 类口误：
  - **重复词/短语**：连续 2+ 个相同词（"就是就是"）→ 保留最后一次
  - **填充词**：嗯、啊、那个、就是 → 默认删除
  - **长停顿**：>1.5 秒的无声段 → 缩短至 0.5 秒
- 填充词列表和停顿阈值可由用户自定义
- 语言支持：MVP 仅支持普通话

**F3: 用户预览与确认**
- 文字稿级别的预览界面（非视频时间线编辑器）
- 口误片段用红色高亮标注，用户点击可恢复保留
- 提供"一键试听"按钮，播放处理后的音频片段
- 用户确认后才进入剪切环节

**F4: 自动加字幕**
- 基于语音识别结果自动生成逐字字幕
- 3 套预设样式：简洁白字 / 黑底白字 / 彩色高亮关键词
- 用户可选：字体大小（小/中/大）、位置（上/中/下）、是否显示字幕

**F5: 视频导出**
- 输出格式：MP4（H.264 + AAC）
- 保持原始分辨率和比例
- 剪切策略：优先无损切割（-c copy）；误差 > 500ms 时回退到重编码
- 音频过渡：crossfade 50ms，避免爆音

**F6: 用户认证与支付**
- 微信扫码登录
- 支付宝/微信支付接入
- 定价模式待用户调研确定（按条 or 包月）

#### P1 — v1.1 计划

- 错误后重说检测（需 LLM 语义理解）
- 多比例导出（竖屏 9:16 + 横屏 16:9）
- OpenClaw Skill 封装
- 更多字幕样式 / 自定义字幕模板

#### P2 — 远期方向

- 风格模板系统（节奏、花字、BGM 自动匹配）
- 课程/教程视频支持（录屏+人脸场景）
- 电商产品视频支持

### 3.3 异常场景

| 场景 | 处理方式 |
|------|---------|
| 上传文件格式不支持 | 前端校验 + 后端二次校验，返回明确错误提示 |
| 文件超过大小/时长限制 | 拒绝上传，提示限制条件 |
| 语音识别质量差（confidence < 0.6 的片段 > 30%）| 告警用户"音频质量较差，识别结果可能不准确，建议重录或手动校对" |
| 处理超时（>10 分钟）| 终止任务，通知用户，建议缩短视频时长 |
| 处理中断（服务器异常）| 保留中间产物，支持从断点恢复 |

## 4. 用户旅程

```
首次使用：
  打开网站 → 微信扫码登录 → 上传视频（支持多文件拖拽）
  → 等待处理（显示进度：识别中/检测中/生成中）
  → 预览文字稿（红色标注口误，点击可恢复）→ 试听
  → 选择字幕样式 → 确认导出
  → 下载成品视频

复购：
  登录 → 上传 → 确认 → 下载（熟练后可跳过预览直接导出）
```

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 10 分钟视频端到端处理 ≤5 分钟（含上传/下载） |
| **并发** | MVP 阶段支持 10 个同时处理任务 |
| **可用性** | 99% uptime（单服务器，非 HA） |
| **数据安全** | 视频处理完成后 24 小时内自动删除；上传页面明确告知数据处理方式 |
| **隐私** | 不存储用户视频用于训练；使用 Whisper API 时音频发送至 OpenAI |

## 6. 输入输出规格

**输入：**
- 视频格式：MP4, MOV, WebM
- 分辨率：≤1080p
- 文件大小：单文件 ≤500MB
- 时长：单文件 ≤30 分钟
- 语言：普通话（方言为已知局限，不保证效果）

**输出：**
- 格式：MP4（H.264 视频 + AAC 音频）
- 分辨率：与输入一致
- 比例：与输入一致（MVP 不做比例转换）

## 7. 验收标准

- [ ] 用户可上传 1~N 个视频文件并排序合并
- [ ] 系统自动识别语音并生成带时间戳的文字稿
- [ ] 系统自动检测重复词、填充词、长停顿并标记
- [ ] 用户可在文字稿界面预览、恢复误删片段、试听
- [ ] 确认后系统自动剪切视频，音频过渡无爆音
- [ ] 用户可选择字幕样式并烧录到视频
- [ ] 用户可下载 MP4 成品
- [ ] 微信登录和支付流程正常
- [ ] 异常场景（格式错误/超限/识别率低/超时）有明确提示
